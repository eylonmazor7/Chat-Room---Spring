In this exercise we have been asked to build some sort of a chat website.
The welcome page will contain a text box where the user will enter its username.
Every try to put a URL which is wrong or belong to a different page will cause a redirecting to the login page.
After entering a username which is not currently in use, the user will be forwarded to the chat page.
The chat will present the last five messages sent by the users. The page will also contain a list of current online users.
An inactive user (for 600 seconds) will be automatically disconnected and sent back to login page.

The last five messages and the active users list will be updated every ten seconds.
Every user could send a message at the box displayed at the page.

There will be a link to the search page, which will contain two searching options- by username and by text.
We have used Thymeleaf to update the html page when loading the page.
There's a use of an SQL database for storing and pulling data (messages and usernames).

Two kinds of Beans took place in our project -
1. Application level Bean- used to manage the list of users and update it. Will be created only once, when the application start its running.
2. Session level Bean- used to represent a user.

  • My Java packages:
    1. Package beans

      Class UserBeansA - this class represent session scope of each user that login. In this session, we save all the data about the user.

      Class UserOnlineA - this class represent application scope. In this class we manage the users status, and save the online users in map with the last time they do some action (after 60 seconds inactivity the user will be disconnected).

    2. Package Controller

      Class ApiController - this controller responsible to update the DB, and send update to the users of who still online and the last messages.

      Class Errors - The controller catches wrong URLs.

      Class MainController - this controller responsible to manage the user navigation between pages, and get new users.

    3. Package DB

      Class ChatMessage - This class represent the our SQL table. The Entity annotation represent work with a database.

      Interface ChatInter - this interface allow us to retrieving data from the data base using queries.

    4. Package HttpHelpers

      Class ReqMes - This class represent every user request for a text search.

      Class ResMes - This class represent our response message for the client.

    5. Bean Configuration A configuration file that determines which configuration level will be assigned for the beans.

  • My Java Scrypt files:
    1. chat.js - Managing data related to messages and updating the user's data.
    2. login.js - Validating the user name entered before moving into the chat page.
    3. search.js - Managing the searches for required words and presenting results.
